# Vue H5/PWA 迁移

## Goal

将原 Flutter Android 原型迁移为面向手机浏览器的 Vue 3 H5/PWA 应用。新版本保留已经验证的家庭奖励闭环、儿童友好体验和视觉资产，并成为后续功能开发的唯一实现；完成行为验证后移除旧 Flutter/Android 工程。

## Background

- 原 Flutter 原型曾覆盖核心闭环：孩子提交任务、家长审核、发放积分和成长值、孩子申请奖励、家长审核兑换、记录积分流水。
- 产品继续使用单孩子、本机持久化、4 位家长 PIN，不依赖服务端。
- 可直接复用的视觉资产包括宠物房背景和团团的普通、开心、低活力、升级状态图片。
- 用户已确认目标技术栈为 Vue 3、TypeScript 和 Vite，迁移完成后仓库只保留 H5/PWA 实现。
- 外部项目仅用于借鉴产品交互和工程组织，不直接 Fork 或复制其实现。

## Requirements

### R1. Web 应用基础

- 仓库根目录直接作为唯一的 H5/PWA 应用工程。
- 使用 Vue 3、TypeScript、Vite、Pinia 和 Vue Router。
- 使用竖屏移动端优先的响应式布局，同时保证常见桌面浏览器可正常使用。
- 支持安装为 PWA，核心页面和本地数据在离线状态下可用。
- H5 质量门通过后删除 Flutter/Android 源码、配置、缓存和旧测试。

### R2. 业务闭环等价迁移

- 迁移宠物房、今日任务、奖励商店、图鉴占位、家长 PIN、家长中心、任务审核、兑换审核和积分记录页面。
- 孩子提交任务后必须进入待审核状态，家长批准前不得发放积分或成长值。
- 家长批准任务后必须更新积分、成长值、心情值、活力值和积分流水。
- 奖励申请必须检查可用积分并冻结积分；批准后正式扣除，驳回后释放冻结积分。
- 家长可以新增和启停任务模板、新增和上下架奖励。
- 默认演示 PIN 保持为 `2580`。

### R3. 数据模型与状态管理

- 使用 TypeScript 显式定义产品 `AppState` 领域类型。
- Pinia store 负责应用状态和领域动作，页面组件不得直接修改持久化对象。
- 领域动作需保持原型中的业务不变量，包括余额计算、重复审核保护、状态转换和流水生成。
- 日期持久化使用 ISO 8601 字符串，加载后在统一边界完成解析。

### R4. 本地持久化与恢复

- 首版保持单设备、本地优先，不引入登录、服务端或云同步。
- 通过 Repository 接口隔离状态管理和浏览器存储。
- 使用 IndexedDB 保存结构化应用状态；首次无数据时加载种子数据。
- 存储记录必须包含 schema version，为后续迁移预留入口。
- 支持导出 JSON 备份和从 JSON 恢复；恢复前要求家长 PIN 并明确提示覆盖风险。

### R5. 视觉与儿童体验

- 复用现有宠物房背景和团团状态图片，不重新生成首版主视觉。
- 宠物房首页保持宠物和房间为第一视觉焦点，不能退化成普通仪表盘。
- 孩子端沿用温和文案，不使用惩罚、财务化或制造焦虑的表达。
- 核心状态不能仅依赖颜色表达；交互目标需适合触屏。
- 点击宠物、任务提交、审核通过和兑换申请必须提供明确反馈。

### R6. 质量与可维护性

- 领域逻辑、Repository 和关键 store action 需要自动化测试。
- 页面关键流程需要组件或端到端测试覆盖。
- `npm run build`、类型检查和测试必须通过。
- TypeScript 实现是业务行为和数据模型的唯一代码真相源。

### R7. 宠物图鉴与切换

- 将图鉴占位页升级为完整宠物目录，展示团团和新增的四只宠物。
- 每只宠物包含普通、开心、兴奋和低能量四种图片状态。
- 宠物按孩子等级逐步解锁，锁定状态必须显示明确的等级条件和进度。
- 已解锁宠物可以在图鉴中切换，选择结果需持久化并同步到宠物房全部表情状态。
- 现有本地数据和旧版 JSON 备份必须自动补齐当前宠物标识，不要求用户重置数据。
- 宠物资产使用描述性、统一的 kebab-case 文件名，不保留 UUID 或尺寸后缀。

## Acceptance Criteria

- [x] 仓库根目录可通过标准 npm 命令安装、启动、测试和构建。
- [x] 手机尺寸打开后默认进入宠物房，并正确显示现有背景、团团、孩子资料、等级和积分。
- [x] 孩子可提交任务，家长通过 PIN 进入审核并批准或驳回，批准后数据和流水正确更新。
- [x] 孩子可申请奖励，冻结、批准扣除和驳回返还逻辑正确，且不能超额申请。
- [x] 家长可新增和启停任务、新增和上下架奖励。
- [x] 刷新页面或关闭浏览器后，应用状态可从 IndexedDB 恢复。
- [x] 家长可导出备份并通过 PIN 验证后恢复备份。
- [x] PWA 可安装，断网后仍能进入核心页面并读取本地数据。
- [x] 常见手机宽度下无横向溢出、文字遮挡或不可点击控件。
- [x] Flutter/Android 工程、配置、缓存和旧测试已删除，README 仅描述 H5/PWA。
- [x] 图鉴展示 5 只宠物、解锁条件和当前选择，达到等级后可切换并在刷新后保留。
- [x] 宠物房会根据当前宠物展示对应的普通、开心、兴奋和低能量图片。

## Out Of Scope

- 用户账号、云同步和多设备协作。
- 多孩子账户、排行榜和家庭社交。
- 真 3D 宠物、复杂装扮系统和 AI 对话。
- 将已退役原生应用的本地数据自动导入浏览器。
- 重新引入或发布原生 Android/Flutter 包。

## Technical Constraints

- 技术栈固定为 Vue 3 + TypeScript + Vite。
- 首版采用本地优先 PWA，Repository 边界为未来远端实现保留替换空间。
- 领域模型、状态动作和持久化契约均由 TypeScript 实现维护。
